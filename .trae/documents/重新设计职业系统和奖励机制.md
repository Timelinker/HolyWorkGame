# 重新设计职业系统和奖励机制

## 实现计划

### 1. 修改职业系统数据结构
- 在 `simple_player_attributes.gd` 中添加新的数据结构来存储职业等级相关信息
- 创建职业等级配置表，包含每个等级的：
  - 升级所需经验值
  - 单次工作获得经验值
  - 单次工作获得金钱数
  - 职业称号

### 2. 重写职业称号函数
- 修改 `get_job_title()` 函数，使用新的职业等级配置表
- 确保正确返回对应等级的职业称号

### 3. 重写工作逻辑
- 修改 `work()` 函数，根据当前职业等级获取对应的：
  - 经验值奖励
  - 金钱奖励
- 更新升级逻辑，使用表格中定义的升级所需经验值

### 4. 调整属性初始化
- 更新初始属性值以匹配表格中的等级 0 数据
- 确保职业等级从 0 开始（公司新人）

### 5. 验证实现
- 确保所有等级的配置正确加载
- 验证工作奖励机制按表格执行
- 验证升级机制正确触发

## 技术实现要点

- 使用数组或字典存储职业等级配置表
- 实现等级查找逻辑，确保正确获取对应等级的配置
- 保持原有信号系统不变，确保UI更新正常
- 保持体力消耗机制不变（每次工作消耗 5 点体力）

## 预期效果

- 职业等级从 0 到 20 按表格定义
- 每次工作获得的经验值和金钱数根据当前职业等级动态变化
- 升级所需经验值按表格定义
- 职业称号正确显示对应等级的名称
