# 修复食物列表不显示的问题

## 1. 问题分析

### 主场景中的设置
- FoodList 节点在 main_scene.tscn 中位置设置为 (20, 100)，大小为 260x140
- 这应该是在屏幕可见区域内的

### 食物列表场景中的设置
- FoodList 节点在 food_list.tscn 中锚点设置为 3（右下角）
- 偏移量设置为 offset_left = -272.0, offset_top = -144.0
- 这些设置可能会覆盖主场景中的位置设置
- 导致食物列表在屏幕外显示

## 2. 解决方案

### 修改 food_list.tscn 文件
- 调整 FoodList 节点的锚点设置
- 将锚点从 3（右下角）改为 0（左上角）或 15（全屏）
- 调整偏移量，确保它在屏幕可见区域内

### 验证修复
- 确保 FoodList 节点在 main_scene.tscn 中正确显示
- 确保食物列表中的内容（如食物项和购买按钮）也能正确显示

## 3. 技术细节

### 锚点系统
- 锚点 0（左上角）：节点位置相对于屏幕左上角
- 锚点 3（右下角）：节点位置相对于屏幕右下角
- 锚点 15（全屏）：节点会自动适应屏幕大小

### 偏移量
- 当锚点设置为 3（右下角）时，负的偏移量会使节点向左上方移动
- 当锚点设置为 0（左上角）时，正的偏移量会使节点向右下方移动

## 4. 预期效果

- 食物列表在屏幕的左上角可见
- 显示所有食物项（浏阳蒸菜、789外卖、日料）
- 每个食物项都有名称、效果和价格显示
- 购买按钮可以正常点击
